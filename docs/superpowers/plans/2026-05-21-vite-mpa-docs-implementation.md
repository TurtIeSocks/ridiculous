# Vite MPA Docs Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reorganize the 238-line `src/app.tsx` monolith into a Vite multi-page app with one URL per component, a sticky sidebar/mobile drawer, custom 404, and codegenned nav from `registry.json`. Static-only output deployable as-is to GitHub Pages.

**Architecture:** Vite MPA with `root: "pages"` and one HTML entry per page (index, color-picker, gradient-editor, unit-input, easing-picker, 404). Each entry boots its own React tree via a `main.tsx` that mounts a page component from `src/pages/*/page.tsx`. Shared layout chrome lives in `src/components/layout/`. Navigation data is generated from `registry.json` to `src/generated/nav.ts` via `scripts/build-nav.mjs` (gitignored, regenerated by `predev`/`prebuild`/`pretypecheck` hooks). Mobile nav uses shadcn's `Sheet` (Radix Dialog).

**Tech Stack:** React 19, Vite 8, TypeScript 6, Tailwind v4 (via `@tailwindcss/vite`), shadcn/ui (button, popover, input, sheet), Biome, Vitest.

**Spec:** [docs/superpowers/specs/2026-05-21-vite-mpa-docs-design.md](../specs/2026-05-21-vite-mpa-docs-design.md)

**Notes for the implementer:**
- The visual chrome is heavily inherited from the current `src/app.tsx`. Reuse class strings verbatim where called for — don't reinterpret. Mesh background, gradient text, glass cards, etc., are project conventions defined in `src/index.css` (see `--color-violet-glow`, `--color-pink-glow`, `bg-mesh`, `bg-grid`, `text-gradient`, `glass-card`).
- Tests in `tests/**` cover component logic, not the docs site. They should stay green throughout. Run `pnpm test` between phases (after Task 7 and Task 17).
- **Do not run the full test suite mid-task.** Run only the tests that are relevant to what you changed. Phase-boundary full runs only.
- Lint + typecheck + tests can run in parallel (single Bash batch). They are read-only of each other's state.
- Some tasks have no meaningful unit-test surface (pure visual composition). Those tasks verify via dev server + manual smoke check, noted explicitly per task.
- Commit after each task.

- [ ] **Step 1: Write the codegen script**

Create `scripts/build-nav.mjs`:

```js
import { readFileSync, writeFileSync, mkdirSync } from "node:fs"
import { dirname, resolve } from "node:path"
import { fileURLToPath } from "node:url"

const __dirname = dirname(fileURLToPath(import.meta.url))
const registryPath = resolve(__dirname, "../registry.json")
const registry = JSON.parse(readFileSync(registryPath, "utf8"))

const items = registry.items.map((it) => ({
  name: it.name,
  title: it.title,
  description: it.description,
  href: `/${it.name}/`,
}))

const out = `// AUTO-GENERATED by scripts/build-nav.mjs. Do not edit.
export interface NavItem {
  name: string
  title: string
  description: string
  href: \`/\${string}/\`
}

export const NAV = ${JSON.stringify(items, null, 2)} as const satisfies readonly NavItem[]
`

const outDir = resolve(__dirname, "../src/generated")
mkdirSync(outDir, { recursive: true })
writeFileSync(resolve(outDir, "nav.ts"), out)

console.log(`Wrote ${items.length} nav items to src/generated/nav.ts`)
```

`NavLink` accepts an `href` (already prefixed with the base path by the caller) and renders an `<a>`. It compares its `href` against `window.location.pathname` to apply an active state.

- [ ] **Step 1: Write the failing test**

Create `tests/nav-link.test.tsx`:

```tsx
import { describe, expect, it } from "vitest"
import { NavLink } from "@/components/layout/nav-link"

describe("NavLink", () => {
  it("renders the children inside an anchor with the given href", () => {
    render(<NavLink href="/ridiculous/color-picker/">Color Picker</NavLink>)
    const link = screen.getByRole("link", { name: "Color Picker" })
    expect(link).toHaveAttribute("href", "/ridiculous/color-picker/")
  })

  it("marks the link active via data-active when href matches current pathname", () => {
    window.history.pushState({}, "", "/ridiculous/color-picker/")
    render(<NavLink href="/ridiculous/color-picker/">Color Picker</NavLink>)
    expect(screen.getByRole("link")).toHaveAttribute("data-active", "true")
  })

  it("does not apply the active state when pathname differs", () => {
    window.history.pushState({}, "", "/ridiculous/gradient-editor/")
    render(<NavLink href="/ridiculous/color-picker/">Color Picker</NavLink>)
    expect(screen.getByRole("link")).toHaveAttribute("data-active", "false")
  })
})
```

- [ ] **Step 3: Write minimal implementation**

Create `src/components/layout/nav-link.tsx`:

```tsx
import type { ReactNode } from "react"
import { cn } from "@/lib/utils"

interface NavLinkProps {
  href: string
  children: ReactNode
  className?: string
}

export function NavLink({ href, children, className }: NavLinkProps) {
  const isActive =
    typeof window !== "undefined" && window.location.pathname === href
  return (
    <a
      href={href}
      data-active={isActive}
      className={cn(
        "block rounded-md px-3 py-2 text-sm transition",
        isActive
          ? "bg-white/10 text-foreground"
          : "text-muted-foreground hover:bg-white/5 hover:text-foreground",
        className,
      )}
    >
      {children}
    </a>
  )
}
```
